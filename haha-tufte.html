[h2 Sidenotes & Margin Notes ] [ [span,newthought One of the most distinctive features ] of Tufte’s style is his extensive use of sidenotes [SN,sn-example-sidenote This is a sidenote. Sidenotes are like footnotes, except they don’t force the reader to jump their eye to the bottom of the page. ]. On sufficiently large viewports, Tufte CSS uses the margin for sidenotes, margin notes, and small figures. On smaller viewports, elements that would go in the margin are hidden until the user toggles them into view. ] [ haha uses JavaScript to facilitate the use of sidenotes and margin notes. The JavaScript (part of [i haha.js ]) produces HTML which is equivalent to the HTML produced by following haha-code, which follows step by step the sidenote-HTML of [em Tufte CSS ]. ] 

[ text ...
[label,margin-toggle_sidenote-number,for=sn-example-sidenote ]
[input,type=checkbox,class=margin-toggle,id=sn-example-sidenote ] 
[span,sidenote ...text of sidenote ...]
... more text ]
[ But with [i haha.js ] this can be reduced to: ] 

[ text ... 
[SN,sn-example-sidenote ...text of sidenote ... ] 
... more text ]
[ Same procedure for a margin note, now using tag [code MN ]. ] 

[ text ... 
[MN,test-mn-haha ...text of margin note ... ] 
... more text ]
[ Demo (change width of browser screen, to see the effect) [MN This is a margin note. Notice there isn't a number preceding the note. ]. On large screens, a margin note omits the reference number. On small screens, a margin note is like a sidenote except its viewability-toggle is a symbol rather than a reference number. haha uses the symbol [span,margin-toggle ⁺ ]. Liepmann uses ⊕. We've tried creating some similarity between numbered and anonymous notes. ]
[h2,i=epigraphs Epigraphs ] [div,epigraph [blockquote [ For a successful technology, reality must take precedence over public relations, for Nature cannot be fooled. ] [footer Richard P. Feynman, [cite "What Do You Care What Other People Think?" ] ] ] [blockquote I do not paint things, I paint only the differences between things. [footer Henri Matisse, [cite Henri Matisse Dessins: thèmes et variations ] (Paris, 1943), 37 ] ] ] [ If you’d like to introduce your page or a section of your page with some quotes, use epigraphs. These are [code blockquote ] elements, children of a block [code \[div,epigraph \] ]. The source goes of a quote goes in a [code footer ] element inside the [code blockquote ]. Following haha-markup implements shorter and longer quotes (with and without a paragraph tag) and shows how multiple quotes within an epigraph fit together with the use of wrapper class [code epigraph ]. ] 

[div,epigraph 
    [blockquote
        [ ... the actual shorter quote ... ] 
        [footer Source of the quote ]
    ]
      
    [blockquote 
      ... the longer quote (no paragraph) ...
      [footer Source of quote ] 
    ]
]
[ Next: a blockquote, without epigraph-wrapper: ] [blockquote [ \[It is\] notable that the Feynman lectures (3 volumes) write about all of physics in 1800 pages, using only 2 levels of hierarchical headings: chapters and A-level heads in the text. It also uses the methodology of [em sentences ] which then cumulate sequentially into [em paragraphs ], rather than the grunts of bullet points. Undergraduate Caltech physics is very complicated material, but it didn’t require an elaborate hierarchy to organize. ] [footer [a,h=https://www.edwardtufte.com/tufte/ Edward Tufte, forum post, 'Book design: advice and examples' thread ] ] ]

Don’t wrap figures in a paragraph tag. Any label or margin note goes in a regular margin note inside the figure. E. Ashworth Underwood, 337-356. From page 71 of Edward Tufte’s [em Visual Explanations ]. ] But tight integration of graphics with text is central to Tufte’s work even when those graphics are ancillary to the main body of a text. In many of those cases, a margin figure may be most appropriate.
